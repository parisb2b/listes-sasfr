1. STACK TECHNIQUE CHOISIE
================================================================================

Stack : HTML/CSS/JS vanilla avec Firebase Compat SDK (CDN) + modules ES
→ Cohérent avec Comptes Client (total.sasfr.com), pas de build step lourd.

Détail :
- Firebase SDK v10.12.0 (compat, namespaced) via CDN gstatic.com
- Firestore (base sasfr-chantiers, collections dédiées listes_lists + users)
- Firebase Authentication (Email/Password + Google + mot de passe oublié)
- SheetJS (xlsx) v0.20.1 via CDN pour les exports Excel
- jsPDF v2.5.1 via CDN pour les exports PDF
- Pas de npm, pas de node_modules, pas de build step
- Déploiement Vercel static (@vercel/static)
- Firestore Security Rules déployées via firebase CLI

Justification : simplicité, cohérence avec l'écosystème SASFR existant,
déploiement instantané, zéro dépendance serveur.

5. MÉCANISME DE BOOTSTRAP ADMIN
================================================================================

Email admin bootstrap : parisb2b@gmail.com

Fonctionnement :
- À l'inscription, le code vérifie si la collection users est vide (premier user)
  OU si l'email correspond à parisb2b@gmail.com → admin=true automatiquement
- Les utilisateurs suivants sont créés avec admin=false par défaut
- Un admin peut promouvoir d'autres utilisateurs via Gestion listes → Membres

Pour attribuer le rôle admin à un autre compte (sans passer par l'UI) :
- Manuellement dans la console Firebase : modifier le doc users/{uid} → admin: true
- Ou via l'UI Gestion listes → Membres → toggle Admin

9. CORRECTIONS POST-DÉPLOIEMENT (2026-06-17 — session de vérification)
================================================================================

Correction 1 — vercel.json : routage qui cassait les fichiers statiques
  Problème : la règle "routes": [{"src": "/(.*)", "dest": "/index.html"}] redirigeait
  TOUTES les requêtes vers index.html, y compris firebase-config.js → le navigateur
  recevait du HTML au lieu de JavaScript → auth undefined → app cassée.
  Correctif : "builds": [{"src": "**/*", "use": "@vercel/static"}] + suppression routes.
  + firebase-config.js inliné directement dans index.html.

Correction 2 — setLang() : bascule FR/中文 qui écrasait le mode inscription
  Problème : setLang() met à jour textContent de TOUS les éléments [data-fr], y compris
  #auth-submit et #auth-toggle. Si on était en mode "S'inscrire", le texte revenait à
  "Se connecter" après un changement de langue.
  Correctif : après le forEach, restaurer les textes si authMode === 'register'.

Correction 3 — onSnapshot : requêtes sans filtre rejetées par Firestore
  Problème : listsRef.onSnapshot() sans filtre → Firestore rejette toute la requête
  car l'utilisateur non-admin n'a pas accès à TOUTES les listes.
  Correctif : pour non-admin, deux onSnapshot filtrés (where createdBy + where
  assignees array-contains) avec cache fusionné. Pour admin, onSnapshot complet.

Correction 4 — firestore.rules : get() sur document users inexistant
  Problème : un get() sur users/{uid} échoue si le document n'existe pas encore
  (utilisateur nouvellement inscrit) → toutes les règles qui dépendent de isAdmin
  échouent → erreur "Missing or insufficient permissions".
  Correctif : fonction helper isAdmin() avec exists() avant get().
  + refactorisation avec fonctions helpers (isAdmin, canAccessList).
